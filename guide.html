1. Installation

Prerequisites

Before you begin, make sure you have:

  • Docker Desktop installed (includes docker and docker-compose)
  • Git installed on your system
  • 4GB+ RAM available for Docker containers
  • A code editor (VS Code recommended)

One-Command Installation

curl -s https://lara-anchor.netlify.app/install.sh | bash

This downloads and extracts the latest Lara Anchor template automatically.

Step-by-Step Setup

1

Clone the Repository

git clone https://github.com/yourusername/lara-anchor.git
cd lara-anchor
2

Create Environment File

cp .env.example .env

Update the database credentials in .env

3

Start Containers

docker-compose up -d

This starts PHP 8.5, MySQL 8.3, and optional Node 22 containers.

4

Install Dependencies

docker-compose exec app composer install
5

Generate Application Key

docker-compose exec app php artisan key:generate
6

Run Database Migrations

docker-compose exec app php artisan migrate

✓ Done! Your Laravel application is now running on http://localhost:8000

3. Configuration

Environment Variables (.env)

Key environment variables for Docker configuration:

APP_MEM_LIMIT=256m # Memory limit for PHP container
DB_MEM_LIMIT=512m # Memory limit for MySQL
MYSQL_DATABASE=laravel # Database name
MYSQL_USER=laravel # Database user
MYSQL_PASSWORD=secret # Database password
MYSQL_ROOT_PASSWORD=root # MySQL root password

⚠️ Note: Always use strong passwords in production. Update MYSQL_PASSWORD and MYSQL_ROOT_PASSWORD.

4. Common Commands

Start Containers

docker-compose up -d

Stop Containers

docker-compose down

View Logs

docker-compose logs -f app

Run Artisan Commands

docker-compose exec app php artisan <command>

Run Composer

docker-compose exec app composer <command>

Access Database

docker-compose exec mysql mysql -u laravel -p laravel

Rebuild Containers

docker-compose up -d --build

Run Node Commands

docker-compose --profile dev up -d
docker-compose exec node npm install

5. Development Workflow

File Synchronization

Your Laravel project files are mounted as volumes. Changes are instantly reflected in the container without rebuilds.

Debug Mode

Enable debug mode in your .env file:

APP_DEBUG=true

Database Access

Connect to MySQL using a GUI client or command line:

  • Host: localhost
  • Port: 3306
  • User: laravel (or root)
  • Password: secret (check .env)
  • Database: laravel

6. Troubleshooting

Port Already in Use

Problem: Port 8000 is already in use.

Solution: Change the port in docker-compose.yml or kill the process:

lsof -i :8000
kill -9 <PID>

Database Connection Error

Problem: Can't connect to MySQL.

Solution: Ensure MySQL container is running:

docker-compose ps

Permission Denied Errors

Problem: Cannot write to storage/ or bootstrap/ directories.

Solution: Run the container with proper permissions:

docker-compose exec app chmod -R 777 storage bootstrap/cache

7. Frequently Asked Questions

Q: Why no Nginx?

A: Laravel's built-in development server (php artisan serve) is sufficient for development and removes unnecessary complexity. For production, you'd want Nginx, but that's not the focus of this lightweight starter.

Q: Why no Redis?

A: We've removed Redis to keep the setup lightweight. If you need caching or queues, you can easily add it to docker-compose.yml. Most Laravel apps don't require it in development.

Q: Can I use this in production?

A: This is designed for development. For production, consider adding Nginx, a reverse proxy, proper logging, and security configurations.

Q: How do I add Redis if I need it?

A: Add a Redis service to docker-compose.yml and configure Laravel's CACHE_DRIVER and QUEUE_CONNECTION in .env

Q: Can I modify memory limits?

A: Yes! Update APP_MEM_LIMIT, DB_MEM_LIMIT in .env, or edit the mem_limit values in docker-compose.yml directly.
